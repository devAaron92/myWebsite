From MIDI to Sound: Building a Tone Generator Step-by-Step

By Aaron Shippey | July 13, 2025

I recently built a tone generator—my first real dive into making my own JSFX plugin. It's a fundamental exercise, but it proves the core concept behind all digital sound: simulation.

A **tone generator** is simple in concept: it’s a type of **oscillator**. If you’re unfamiliar with that term, just think of something that repeats in cycles—kind of like a pulse or a wave. The complexity of any synthesizer comes down to how accurately and efficiently it runs these cycles.

Step 1: User Input — Picking a Note (MIDI)

First, the user picks a note using a slider or MIDI controller. This note is received in **MIDI format**, which is just a number between 0 and 127. For example, 60 is Middle C, 69 is A4 (A above middle C), etc. However, MIDI numbers aren’t in Hertz (the unit we use for frequency), so we need to **convert** that data.

Step 2: Convert MIDI to Frequency (Hertz)

To turn that MIDI note into a real pitch, we use a simple exponential formula. This formula finds the target pitch by shifting up or down from a standard reference frequency, usually $\mathbf{440 Hz}$ (the pitch of A4).

$\text{frequency} = 440 \times 2^{\left(\frac{\text{MIDI} - 69}{12}\right)}$

So if the user selects MIDI note 60 (C4), the formula figures out the vibrating speed we want—in this case, approximately $\mathbf{261.63 Hz}$. That’s the actual rate of vibration.

Step 3: Calculate Rotation Speed (Angular Velocity)

Now that we know the frequency, we calculate how fast our waveform should move forward—this is called **angular velocity ($\omega$)**. It tells us how quickly we spin through one full wave cycle (a circle). This is crucial because it’s the number we use for every sample.

$\omega = 2 \times \pi \times \frac{\text{frequency}}{\text{sampleRate}}$

This calculation lets us figure out how much to advance our pointer each time the computer processes a tiny chunk of audio (a **sample**). Most modern systems process audio at a **sample rate** of $\mathbf{44,100}$ samples per second.

Step 4: Keep Track of Where We Are — Phase

To know what part of the wave we’re at, we use a variable called **phase ($\phi$)**. Think of it like a pointer moving around a circle. Every single sample, we move the phase forward just a little bit by adding the angular velocity ($\omega$):

$\phi += \omega$

This is how we simulate time passing in very small, consistent steps. **Instability and sync bugs often happen here**—if the phase calculation is not sample-accurate across different DAWs, the sound will break.

Step 5: Create the Sound (Sine Function)

Finally, we take the current phase ($\phi$) and pass it into a sine function:

$\text{output} = \sin(\phi)$

That simple math gives us a perfectly smooth wave—a **sine wave**—which is the sound! The faster we spin through the wave (higher frequency), the higher the perceived pitch.

So What’s Really Happening?

  • The user picks a note (MIDI number).
  • We figure out how fast that note vibrates (Hertz).
  • We simulate that vibration by looping through tiny audio samples.
  • We create the wave by calculating points on a circle (a sine wave).

It sounds like a lot, but it’s really just a few numbers being added together over and over again. Understanding this core loop is the key to both **building**—and more importantly—**breaking and testing** audio software efficiently.
