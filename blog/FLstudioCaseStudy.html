FL Studio Exploratory Testing Case Study

By Aaron Shippey

This case study documents my first experience with FL Studio, approached from the perspective of a plugin QA tester. My goal was to understand FL Studio's core workflow well enough to test plugins effectively in this environment.

Section 1: Introduction

Rather than following tutorials initially, I explored FL Studio on my own. I documented what felt intuitive, where friction appeared, and how FL Studio's workflow philosophy differs from track-based DAWs. This case study focuses on four core areas: plugin and asset loading, FL Studio's pattern-based workflow, MIDI recording.

Section 2: First impressions

One feature I really appreciate in FL Studio is the hint panel in the top-left corner, just below the Escape and Maximize icons. Whatever element your mouse hovers over, the panel displays a short description of that tool or control.

It’s like having an integrated reference while exploring the DAW.

Another feature that stood out immediately was how customizable the FL Studio interface is. Like in a physical studio, where your mixing console isn’t glued to a certain place, FL Studio allows you to resize, reposition, and organize nearly every panel.

After getting a feel for the general workflow, I wanted to find a plugin and open it. As a new user, I found it was clear locating the plugin type I wanted.

After I clicked on the browser icon, I found the plugin folder was organized clearly. I immediately knew how to find the plugin I was looking for. There’s the plugin database with ‘Effects’ and ‘Generator’ folders.

Section 4: Playlist

After several days of exploring FL Studio, I still haven't successfully recorded MIDI from my keyboard. This is my biggest friction point so far.

Here's what I've tried:

  • I opened the Channel Rack, Mixer, and Playlist simultaneously.
  • I verified my MIDI keyboard was working (I could hear notes when I played). I checked my settings: both 'Enable MIDI remote control' and 'Enable MIDI output' were enabled.
  • In the Channel Rack, I selected the channel I wanted to record to. In the Mixer, I clicked 'arm disk recording' for the appropriate channel. Then I pressed the Record button in the top menu and played on my keyboard.
Result: No MIDI was played on playback. I've repeated this process multiple times with different approaches, selecting different channels, trying different mixer settings, enabling/disabling various options — but I haven't been able to capture MIDI input from my keyboard.
What works: I can create MIDI patterns in the step sequencer. I can draw MIDI notes in the Piano Roll. I can record audio without issues.
What I can't figure out: How to record MIDI in real-time from my keyboard, the way I would in Logic or REAPER — arm a track, press record, play, and have the performance captured. This could be user error on my part, a hardware/driver issue, or a gap in my understanding of FL Studio's recording workflow. This may suggest a UX friction point.

Section 5: Channel rack

When I first explored the Channel Rack, I didn’t realize it functioned differently from tracks in other DAWs. While it lets you manage and sequence all your channels, these channels must be routed to the Mixer to apply FX or mix linearly.

This is where I found I manage all of my channels.

Adding a new channel is simple and straightforward. You can simply click the + icon under the channels where you can add audio clips, MIDI, virtual instruments.

Below the channel list is FL Studio’s step sequencer, one of its most recognizable features of the channel rack. It lets you create beats quickly in a grid-based, pattern-oriented workflow — no need to play in real time or record live performance to get a rhythm going.

Section 6: The mixer

In the mixer console, I noticed the track latency button. I haven’t used this feature before, so I’m experimenting to understand how it affects workflow and timing.

When I initially came to the mixing console, I wasn’t sure how the ‘enable fx’ button worked. It turned out I needed to load a plugin in the corresponding slot.

It also took me a while to successfully load an FX plugin in the Mixer. At first, I went to the Mixer panel, which seemed like the obvious place to add FX, but I didn’t see a plugin option in the pop-up menu, which confused me. After exploring further, I clicked the panel again and discovered a second menu with many more options, including a ‘Load Plugin’ function at the top.

Section 7: Technical Issues Encountered

Format-Specific Crash: Surge XT (VST3)
FL Studio 25.2.3.4889 trial version (Apple Silicon) crashes consistently when applying filters in Surge XT's VST3 version on macOS Sonoma 14.4. Error: EAccessViolation.
However, the Audio Units (AU) version of Surge XT works without issues in the same environment. This suggests a VST3 bridge compatibility issue in FL Studio trial on Apple Silicon, rather than a problem with Surge XT itself.

Section 8: Final thoughts

Exploring FL Studio showed me that a DAW workflow doesn’t need to be fixed — there can be countless ways to approach creating music. One feature I wish FL Studio would allow is minimizing the main window, so it doesn’t always take up the entire screen.
